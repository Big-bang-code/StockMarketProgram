# 财富管家应用产品设计原型 - v1.1版本

## 版本概述

### 版本号
**v1.1 - 财富分类管理版本**

### 版本目标
在v1.0基础版本的基础上，新增财富分类管理功能，提供更详细的资产分析和可视化展示，帮助用户更好地理解个人财富构成和分布。

### 发布时间
预计2026年9月

---

## 新增功能设计

### 1. 财富分类管理

#### 1.1 分类体系
- **股市资产**：股票、股票基金、股票账户余额等
- **公积金**：住房公积金、补充公积金等
- **支付宝**：支付宝余额、余额宝、定期理财等
- **基金**：各类基金产品（货币基金、债券基金、混合基金、股票基金等）
- **存款**：银行存款、定期存款、大额存单等

#### 1.2 分类管理功能
- **分类设置**：允许用户自定义财富分类
- **分类编辑**：支持对现有分类进行编辑和删除
- **分类排序**：可调整分类显示顺序
- **分类图标**：为每个分类设置对应的图标

### 2. 财富总额展示

#### 2.1 表格展示
- **分类明细**：以表格形式显示各类财富的详细数据
- **实时更新**：数据实时同步，确保准确性
- **金额格式化**：支持不同货币单位和显示格式
- **数据导出**：支持将表格数据导出为Excel或CSV格式

#### 2.2 饼状图展示
- **资产分布**：以饼状图直观展示各类财富占比
- **交互式图表**：支持点击图表区域查看详细数据
- **图例说明**：清晰的图例说明，方便用户理解
- **数据钻取**：支持从总览到明细的深度分析

### 3. 资产配置分析

#### 3.1 配置分析功能
- **资产配置建议**：根据用户资产分布提供配置建议
- **风险评估**：评估用户的风险承受能力和资产风险
- **优化建议**：提供资产优化配置方案
- **历史对比**：对比不同时间段的资产配置变化

#### 3.2 配置报告
- **配置报告生成**：自动生成资产配置分析报告
- **报告导出**：支持报告的PDF和Excel格式导出
- **定期报告**：支持定期发送资产配置报告

### 4. 分类统计功能

#### 4.1 分类统计维度
- **时间维度**：按日、周、月、年统计各类资产
- **金额维度**：按金额区间统计各类资产
- **增长维度**：统计各类资产的增长情况
- **对比维度**：对比不同分类之间的统计数据

#### 4.2 统计图表
- **趋势图**：各类资产的增长趋势图
- **对比图**：不同分类之间的对比图表
- **分布图**：各类资产的分布情况图表
- **排行榜**：资产增长和金额的排行榜

## 技术实现

### 1. 前端技术改进

#### 1.1 图表库升级
- **图表库**：升级为ECharts 5.x版本
- **交互功能**：增强图表的交互功能
- **响应式设计**：优化图表在不同设备上的显示
- **性能优化**：提升图表渲染和数据更新性能

#### 1.2 UI组件库
- **表格组件**：升级为高级表格组件
- **表单组件**：优化表单验证和用户体验
- **图表组件**：封装可复用的图表组件
- **布局组件**：优化页面布局和响应式设计

### 2. 后端技术优化

#### 2.1 数据库优化
- **分类字段**：在records表中添加category_type字段
- **索引优化**：为分类查询添加索引
- **查询优化**：优化复杂查询的性能
- **数据同步**：确保数据实时同步和一致性

#### 2.2 API接口优化
- **分类API**：新增分类管理相关API
- **统计API**：优化统计查询接口
- **图表API**：新增图表数据接口
- **报告API**：新增报告生成接口

### 3. 数据模型更新

#### 3.1 记录表优化
```sql
ALTER TABLE records
ADD COLUMN category_type TEXT CHECK(category_type IN ('stock', 'provident_fund', 'alipay', 'fund', 'deposit'));
```

#### 3.2 分类配置表
```sql
CREATE TABLE category_configs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    category_name TEXT NOT NULL,
    category_type TEXT NOT NULL,
    category_icon TEXT,
    sort_order INTEGER DEFAULT 0,
    is_active BOOLEAN DEFAULT 1,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id)
);
```
