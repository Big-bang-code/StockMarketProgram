# 财富金额统计与增长应用产品设计原型

## 产品概述

### 产品名称
**财富管家** - 个人财富金额统计与增长分析应用

### 产品愿景
帮助用户轻松记录、统计和分析个人财富状况，通过直观的数据可视化展示财富增长趋势，实现财富管理的科学化和便捷化。

### 产品定位
面向个人用户的轻量级财富管理工具，专注于财富数据的记录、统计和可视化分析，提供简洁易用的操作界面和强大的数据展示功能。

### 核心价值
- **便捷记录**：快速记录收入、支出和资产变动
- **智能统计**：自动计算财富增长、收支比例等关键指标
- **直观展示**：通过图表可视化财富增长趋势
- **数据分析**：提供多维度的数据洞察和分析功能

## 用户需求分析

### 目标用户
- 个人财务管理需求者
- 关注财富增长的用户
- 需要简单易用的记账工具的用户
- 希望了解个人财务状况的用户

### 用户痛点
1. 传统记账方式繁琐，缺乏直观的数据展示
2. 财富增长趋势不明确，难以评估财务状况
3. 数据分散，难以进行全面的财务分析
4. 需要一个简单易用的工具来管理个人财务

### 用户需求优先级
| 需求类型 | 需求描述 | 优先级 |
|---------|---------|--------|

### 核心功能详细设计

#### 1. 数据录入功能
**功能描述**：允许用户快速记录收入、支出和资产变动

- **收入记录**：
  - 金额
  - 来源分类
  - 日期
  - 备注
- **支出记录**：
  - 金额
  - 用途分类
  - 日期
  - 备注
- **资产记录**：
  - 金额
  - 资产类型
  - 日期
  - 备注

**交互设计**：
- 提供快速添加入口
- 支持键盘快捷键操作
- 自动保存草稿功能
- 分类下拉选择或搜索

#### 2. 数据管理功能
**功能描述**：管理和维护用户的财务记录

- **记录列表**：
  - 按时间倒序展示所有记录
  - 支持分页和搜索
- **编辑记录**：
  - 修改记录的所有字段
  - 保存修改历史
- **删除记录**：
  - 支持单个删除和批量删除
  - 删除前确认提示
- **记录查询**：
  - 按日期范围查询
  - 按分类查询
  - 按金额范围查询

#### 3. 数据分析功能
**功能描述**：对用户财务数据进行统计和分析

- **财富总额计算**：
  - 实时计算当前财富总额
  - 显示每日/每周/每月变化
- **增长率计算**：
  - 计算日/周/月/年增长率
  - 显示同比和环比数据
- **收支分析**：
  - 计算收入和支出总和
  - 分析收支比例
  - 显示盈余/赤字情况
- **分类统计**：
  - 按收入来源分类统计
  - 按支出用途分类统计
  - 按资产类型分类统计

#### 4. 图表可视化功能
**功能描述**：通过图表直观展示财务数据

- **财富增长趋势图**：
  - 折线图展示财富随时间的变化
  - 支持切换时间范围（日/周/月/年）
  - 显示增长率和关键点标注
- **收支对比图**：
  - 柱状图对比收入和支出
  - 支持按月/季度/年对比
- **支出分类图**：
  - 饼图展示各类支出占比
  - 支持自定义分类
- **资产分布图**：
  - 饼图展示各类资产占比
  - 显示资产构成

#### 5. 财务报告功能
**功能描述**：生成详细的财务分析报告

- **日报表**：
  - 每日收支明细
  - 财富变动情况
- **月报表**：
  - 月度收支统计
  - 财富增长分析
- **季度/年度报表**：
  - 季度/年度财务总结
  - 长期增长趋势分析
- **自定义报告**：
  - 支持用户自定义报告内容
  - 导出报告为PDF或Excel格式

## 数据模型设计

### 核心数据模型

#### 1. 用户表（users）
```sql
CREATE TABLE users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    username TEXT NOT NULL UNIQUE,
    password_hash TEXT NOT NULL,
    email TEXT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

#### 2. 记录表（records）
```sql
CREATE TABLE records (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    record_type TEXT NOT NULL CHECK(record_type IN ('income', 'expense', 'asset')),
    amount REAL NOT NULL,
    category TEXT NOT NULL,
    record_date DATE NOT NULL,
    description TEXT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id)
);
```

#### 3. 分类表（categories）
```sql
CREATE TABLE categories (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER,
    category_type TEXT NOT NULL CHECK(category_type IN ('income', 'expense', 'asset')),
    category_name TEXT NOT NULL,
    category_icon TEXT,
    is_default BOOLEAN DEFAULT 0,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id)
);
```

#### 4. 财务目标表（financial_goals）
```sql
CREATE TABLE financial_goals (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    goal_name TEXT NOT NULL,
    target_amount REAL NOT NULL,
    current_amount REAL DEFAULT 0,
    start_date DATE NOT NULL,
    end_date DATE NOT NULL,
    goal_type TEXT CHECK(goal_type IN ('saving', 'investment', 'debt')),
    status TEXT DEFAULT 'active' CHECK(status IN ('active', 'completed', 'cancelled')),
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id)
);
```
